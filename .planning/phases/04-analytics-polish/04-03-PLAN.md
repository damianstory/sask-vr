  <files>next.config.ts, package.json</files>
  <read_first>

  </read_first>
  <action>
    1. Install bundle analyzer: `npm install --save-dev @next/bundle-analyzer`

    2. Update next.config.ts to wrap config with bundle analyzer:
       ```typescript
       import type { NextConfig } from 'next'
       import bundleAnalyzer from '@next/bundle-analyzer'

       const withBundleAnalyzer = bundleAnalyzer({
         enabled: process.env.ANALYZE === 'true',
       })

       const nextConfig: NextConfig = {
         /* config options here */
       }

       export default withBundleAnalyzer(nextConfig)
       ```
       If the ESM import of @next/bundle-analyzer fails during build, try:
       ```typescript
       const withBundleAnalyzer = require('@next/bundle-analyzer')({ enabled: process.env.ANALYZE === 'true' })
       ```

    3. Run production build to verify it succeeds: `npm run build`
       - Check build output for total JS size
       - If build shows JS chunks, sum the main chunks (excluding maplibre-gl lazy chunk) and verify < 150KB gzipped

    4. Run bundle analyzer to verify maplibre-gl is code-split:
       `ANALYZE=true npm run build`
       - This opens a browser treemap or writes to .next/diagnostics/analyze
       - Verify maplibre-gl appears in a separate lazy chunk, NOT in the main client bundle

    5. Verify screen transition performance:
       - Screen transitions use client-side state changes (`setCurrentScreen`) with CSS animations
       - No data fetching or heavy re-renders between screens
       - The key={currentScreen} remount is lightweight (single component)
       - This is inherently fast (<500ms) by architecture -- no fix needed unless build reveals blocking re-renders

    6. If bundle size exceeds 150KB gzipped (excluding maplibre-gl):
       - Check if focus-trap-react or @next/third-parties added significant weight
       - focus-trap-react is ~3KB gzipped, should be negligible
       - @next/third-parties GoogleAnalytics is a thin wrapper, should be negligible
       - If still over budget, investigate treemap for unexpected large dependencies
  </action>
