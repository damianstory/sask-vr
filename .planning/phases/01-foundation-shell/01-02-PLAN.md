4. Create `app/pre-vr/page.tsx` as a client component ('use client'):

**State:**
- `const [currentScreen, setCurrentScreen] = useState<1 | 2 | 3 | 4 | 5 | 6>(1)`
- `const [direction, setDirection] = useState<'forward' | 'backward'>('forward')`
- `const [isInitialMount, setIsInitialMount] = useState(true)` -- to skip animation on first render

**Navigation handlers:**
- `goNext`: if currentScreen < 6, set direction to 'forward', increment screen, set isInitialMount false
- `goPrev`: if currentScreen > 1, set direction to 'backward', decrement screen, set isInitialMount false

**Screen map:**
- `Record<1|2|3|4|5|6, React.ReactNode>` mapping to ScreenOne through ScreenSix components
- Import all six screen components from `./components/ScreenOne` etc.

**Render structure:**
```
<SessionProvider>
  <div className="flex flex-col min-h-screen">
    <ProgressBar current={currentScreen} total={6} />
    <div className="flex-1 relative overflow-hidden">
      <div
        key={currentScreen}
        className={isInitialMount ? '' : direction === 'forward' ? 'animate-slide-left' : 'animate-slide-right'}
      >
        {screens[currentScreen]}
      </div>
    </div>
    <Navigation
      currentScreen={currentScreen}
      totalScreens={6}
      onNext={goNext}
      onPrev={goPrev}
    />
  </div>
</SessionProvider>
```

Key: the `key={currentScreen}` on the animation wrapper triggers React to remount the div on screen change, which restarts the CSS animation. The `isInitialMount` flag prevents the first screen from animating in.
