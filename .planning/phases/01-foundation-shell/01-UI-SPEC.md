# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation + Shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom components with Tailwind CSS) |
| Icon library | Custom SVG (line-based, 2px stroke, Navy + Primary Blue fills) |
| Font | Open Sans via `next/font/google` with `display: swap` |

**Rationale:** DESIGN_SPECS.md and TECH_SPECS.md prescribe a custom Tailwind CSS + CSS custom properties approach. The brand token system is fully defined. No component library needed for Phase 1's scope (landing page, progress indicator, navigation buttons, placeholder shells).

---

## Spacing Scale

Declared values (8px base grid, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon-to-label gaps, tight internal padding |
| `--space-2` | 8px | Element padding, compact margins, gap between adjacent tap targets |
| `--space-3` | 16px | Standard padding inside cards and tiles, mobile gutters and margins |
| `--space-4` | 24px | Section padding, card internal margins, tablet/desktop gutters |
| `--space-5` | 32px | Between sections on a screen |
| `--space-6` | 48px | Between major screen sections, screen top/bottom padding |
| `--space-7` | 64px | Page-level vertical rhythm (desktop only) |

Exceptions: Minimum 44x44px touch targets on all breakpoints, 48x48px on mobile where layout allows. 8px minimum gap between adjacent tappable elements.

**Source:** DESIGN_SPECS.md section 2.3

## Typography

Font: Open Sans (fallback: -apple-system, BlinkMacSystemFont, sans-serif)

| Role | Size (desktop / mobile) | Weight | Line Height | Color |
|------|------------------------|--------|-------------|-------|
| Screen heading | 40px / 28px | 900 (Black) | 1.2 | Navy `#22224C` |
| Subheading | 22px / 18px | 300 (Light) | 1.4 | Neutral-5 `#485163` |
| Body | 16px / 16px | 300 (Light) | 1.75 | Neutral-5 `#485163` |
| Button label | 16px / 16px | 900 (Black) | 1.0 | White `#FFFFFF` (on blue) |
| Label / caption | 14px / 14px | 300 (Light) | 1.5 | Neutral-4 `#65738B` |

**Phase 2+ type scale extension:** Display role (64px/48px, weight 900) will be introduced in Phase 2 when stat numbers and large display text are needed.

**Declared weights (2 total):**
- **900 (Black):** Screen headings, button labels
- **300 (Light):** Subheadings, body text, labels/captions

**Rules:**
- Maximum two font weights per screen: 900 for headings/actions + 300 for body/supporting
- No font size below 14px anywhere
- Headings always Navy; body text always Neutral-5 or Neutral-6

**Source:** DESIGN_SPECS.md section 2.2

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F6F6FF` Off-White | Page backgrounds, content areas |
| Secondary (30%) | `#C6E7FF` Light Blue | Path cards, section highlights, tile default background |
| Accent (10%) | `#0092FF` Primary Blue | Continue buttons, progress dots (filled), selected states, focus rings |
| Text primary | `#22224C` Navy | Headings, primary text, navigation labels |
| Text secondary | `#485163` Neutral-5 | Body text, card subtext |
| Text tertiary | `#65738B` Neutral-4 | Captions, secondary labels, progress indicator text |
| Disabled surface | `#AAB7CB` Neutral-3 | Disabled button backgrounds, hollow progress dots, placeholder text |
| Border default | `#D9DFEA` Neutral-2 | Card borders, subtle separators, placeholder box borders |
| Border subtle | `#E5E9F1` Neutral-1 | Dividers, placeholder box backgrounds |
| Hover accent | `#0070CC` Blue Dark | Hover state on Primary Blue elements |

Accent (`#0092FF` Primary Blue) reserved for:
- Continue / Next buttons (filled background)
- Progress dots (filled state for completed/current screens)
- Current dot pulse animation
- Focus ring on keyboard navigation (3px outline, 3px offset)
- Selected state borders (Phase 2+)

**No destructive actions exist in Phase 1.** Error states use Neutral-4 text on Neutral-1 background with a retry action -- no red or non-brand colors introduced.

**Accessibility constraints:**
- Primary Blue on white: 3.1:1 -- use only for large text (18px+ bold or 24px+ regular) and UI components, never small body text
- Navy on white: 13.5:1 -- passes AA and AAA for all text sizes
- All text-on-color combinations verified via automated contrast checker during implementation

**Source:** DESIGN_SPECS.md sections 2.1 and 6.1

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page title | "Carpentry Career Explorer" |
| Pre-VR card heading | "I'm about to do VR" |
| Pre-VR card subtext | "Learn what carpentry in Saskatchewan is really like." |
| Post-VR card heading | "I just finished VR" |
| Post-VR card subtext | "Reflect on what you just experienced." |
| Progress indicator label | "{N} of 6" (e.g., "3 of 6") |
| Next button label | "Next" |
| Back button label | "Back" |
| Back button (Screen 1) | Disabled -- no backward navigation from first screen |
| Next button (Screen 6) | Hidden or absent -- Screen 6 is the final Pre-VR screen |
| Empty state (placeholder screens) | Static gray boxes with Neutral-1 background and Neutral-2 borders -- no text copy needed for placeholder shapes |
| Error state (page not found) | Not applicable for Phase 1 -- only three routes exist, all are valid |

**Tone:** First-person casual -- "excited older sibling" not "guidance counsellor." Direct, conversational, action-oriented. Avoid brochure language ("Welcome to your learning journey").

**Source:** CONTEXT.md decisions (landing page copy), DESIGN_SPECS.md section 2.5

## Component Inventory (Phase 1)

Components built in this phase that establish visual contracts for later phases:

| Component | Visual Contract |
|-----------|----------------|
| **Path Card (Landing)** | 400x280px desktop, full-width 180px mobile. Light Blue bg `#C6E7FF`. Heading 24px/900/Navy. Subtext 16px/300/Neutral-5. Hover: shadow lift `0 8px 24px rgba(34,34,76,0.1)`, border shifts to Primary Blue. Focus: Primary Blue 3px outline, 3px offset. Semantic `<button>` element. |
| **Progress Bar** | Six dots, 12px diameter each, 8px gap. Filled: Primary Blue `#0092FF`. Hollow: Neutral-3 `#AAB7CB`. Current dot: subtle pulse animation (scale 1.0-1.2, 2s loop). Text label below: "{N} of 6" in 14px/300/Neutral-4. Centered top of screen content area. `role="progressbar"` with `aria-valuenow`. |
| **Navigation** | Back + Next buttons. Next: Primary Blue bg, white text, 16px/900. Back: secondary style (outlined or text-only). Disabled state: Neutral-3 bg, Neutral-4 text. Positioned at bottom of screen content. |
| **Screen Shell (placeholder)** | Correct layout grids and section containers per DESIGN_SPECS screen specs. Placeholder shapes: Neutral-1 `#E5E9F1` background, Neutral-2 `#D9DFEA` borders, `border-radius: 12px`. Structure driven by `carpentry.json` data (number of tiles, steps, pins, etc.). |
| **Slide Transition** | Forward: slide-left (translateX 100% to 0, opacity 0 to 1). Backward: slide-right (translateX -100% to 0, opacity 0 to 1). Duration: 400ms. Easing: ease-out. Skip animation on initial mount. `prefers-reduced-motion`: instant, no animation. |

## Responsive Breakpoints

| Breakpoint | Columns | Gutter | Margin | Max Content Width |
|------------|---------|--------|--------|-------------------|
| < 640px (mobile) | 4 | 16px | 16px | 100% |
| 640-1023px (tablet) | 8 | 24px | 24px | 100% |
| >= 1024px (desktop/Chromebook) | 12 | 24px | auto | 1120px centered |

**Phase 1 layouts:**
- Landing: Cards stacked full-width on mobile, side-by-side on tablet+
- Pre-VR screens: Single-column content, centered, max-width varies by screen (per DESIGN_SPECS 5.1)
- Post-VR: Single-column, centered, max 640px content width

**Source:** DESIGN_SPECS.md sections 2.3 and 5.1
