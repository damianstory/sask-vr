## Spacing Scale

Declared values (multiples of 4, matching existing CSS custom properties in globals.css):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon-to-label gap, tight internal padding |
| `--space-2` | 8px | Element padding, compact margins, gap between adjacent tappable elements |
| `--space-3` | 16px | Standard padding inside cards and tiles, mobile gutters |
| `--space-4` | 24px | Section padding, card internal margins, desktop gutters |
| `--space-5` | 32px | Between sections on a screen, vertical gap between collapsed timeline steps |
| `--space-6` | 48px | Between major screen sections, screen top/bottom padding |
| `--space-7` | 64px | Page-level vertical rhythm on desktop |

Exceptions:
- Touch targets: minimum 44x44px (48x48px on mobile where layout allows), per WCAG AA / A11Y-03
- Timeline step nodes: 40px diameter (48px for "You are here" node)
- Map pin markers: 40px diameter on all breakpoints (meets touch target minimum)
- Observation prompt card gap (Screen 6): 12px between stacked cards. Rationale: 8px is too tight for cards containing multi-line prompt text and an icon, creating a cramped feel; 16px creates excessive vertical space that makes the 3 prompt cards look disconnected rather than forming a cohesive set.
- Checklist row gap (Post-VR): 12px vertical between items. Rationale: 8px is too tight for rows with 24px checkboxes and 16px labels, making tap targets overlap visually; 16px creates too much vertical space, pushing the bottom items below the fold on mobile and making the list feel sparse rather than actionable.

## Screen-by-Screen Interaction Contract

### Screen 1 -- Salary Counter & Hook

**Animation sequence:**
1. Hook question appears immediately as static bold text (no animation)
2. Salary counter starts at $0 and ticks up via odometer-style digit animation over 2000ms with ease-out easing; each digit column rolls independently via CSS `transform: translateY()` with 50ms stagger between digits
3. Stat cards appear in staggered fade-in + slide-up: first card at +300ms after counter lands, then +500ms, +700ms; each slides up 8px as it fades in over 400ms ease-out

**Stat card styling:** Navy (#22224C) background, white text, bold number on top (24px, weight 800), small label below (14px, weight 300), border-radius 12px, padding 16px

**prefers-reduced-motion:** Show final salary number immediately. Stat cards appear without animation (instant).

**Responsive:** Counter 64px desktop / 48px mobile. Stats horizontal 3-up desktop / stacked vertically mobile.

### Screen 2 -- Task Tiles

**Grid:** `grid-cols-2 md:grid-cols-3` (2-col mobile, 3-col desktop)

**Tile states:**
- Unselected: white bg, 1px solid `--myb-neutral-2` border, border-radius 12px
- Hover: subtle shadow lift, border shifts to `--myb-neutral-3`
- Selected: `--myb-light-blue` bg, 2px solid `--myb-primary-blue` border, checkmark badge in top-right corner (Primary Blue circle with white check icon, 24px diameter)
- Active/pressed: scale 0.98

**Tile content:** Emoji on soft colored background circle (48px diameter, centered in top 60% of tile) + label below + description below label

**4th selection attempt:** Tapped tile performs horizontal shake animation (CSS @keyframes, 300ms ease-in-out: 0px -> -4px -> 4px -> 0px) + inline message "You can pick up to 3!" appears below grid in `--myb-neutral-4`, auto-dismisses after 3000ms

**Accessibility:** Each tile is `<button>` with `aria-pressed="true|false"`. Focus ring: 3px solid Primary Blue outline.

### Screen 3 -- Employer Map

**Map:** mapcn (MapLibre GL) component centered on Regina, SK (-104.6189, 50.4452) at fixed zoom ~11. All interaction disabled (no pan/zoom/rotate). Free OpenStreetMap positron-style tiles (light, minimal).

**Code-split:** Screen 3 must use `React.lazy()` + `Suspense` to avoid adding maplibre-gl (~200KB) to initial bundle.

**Pin markers:** 40px diameter circles, `--myb-primary-blue` fill, white pin/map icon centered. Hover: scale 1.1 + shadow. Active: scale 0.95.

**Bottom sheet (mobile) / floating card (desktop):**
- White bg, border-radius 12px, shadow `0 8px 24px rgba(34,34,76,0.12)`
- Width: 320px desktop / 90% viewport mobile
- Content: company name (24px, 800, Navy), description (14px, 300, Neutral-5), employee count (14px, 300, Neutral-4), optional quote (14px, 300, italic, Neutral-4)
- Employer logo placeholder: 48x48 circle, Navy text on Light Blue bg, company initial letter
- Close: X button in top-right with `aria-label="Close employer card"` + backdrop tap + Escape key
- Mobile: slides up from bottom with semi-transparent backdrop overlay
- Desktop: positioned near pin, offset right or left based on pin location
- Animation: scale 0.9->1.0 + fade-in over 250ms ease-out; close: fade-out + scale-down 150ms ease-in

**Accessibility:** Card uses `role="dialog"` with `aria-labelledby` pointing to company name. Focus trapped inside card while open. Pins are `<button>` with `aria-label="View {name}"`. Close button has `aria-label="Close employer card"`.

### Screen 4 -- Career Pathway Timeline

**Layout:** Vertical connected timeline. 2px connecting line on the left side. Circular step nodes connected by line.

**"You are here" node (step 1):**
- 48px diameter, `--myb-primary-blue` fill, white interior
- Pulse ring animation: scale 1.0->1.2->1.0 over 2000ms ease-in-out infinite
- Starts expanded by default

**Future step nodes:**
- 40px diameter, hollow/outlined: 2px border `--myb-neutral-3`, white bg
- Final node (step 5): `--myb-navy-light` (#3A3A6B) fill

**Connecting line:**
- Near-term (steps 1-2): solid 2px `--myb-primary-blue`
- Future (steps 3+): dashed 2px `--myb-neutral-3` (border-left dashed)

**Accordion:** Single `expandedStepId` state. Only one step expanded at a time. Clicking expands or collapses. First step expanded by default.

**Expand/collapse animation:** `grid-template-rows: 0fr / 1fr` technique with `overflow: hidden` on inner wrapper. Transition duration 300ms ease-out (expand), 200ms ease-in (collapse).

**Expanded content fields:**
- Description paragraph (16px, 300)
- Duration badge: `--myb-light-blue` bg pill, Navy text (14px, 800)
- Earnings range: `--myb-light-blue` bg pill, Navy text (14px, 800)
- Program name tags: `--myb-light-blue` bg pill, Navy text (14px, 800)
- Courses listed if present

**Accessibility:** Each step button has `aria-expanded`. Collapsed content uses `overflow: hidden` (not `display: none`) so grid transition works.

### Screen 6 -- VR Prep

**Layout:** Single column, max 640px centered. Read-only, no required interactions.

**Heading:** 40px desktop / 28px mobile, Navy, weight 800

**Observation prompt cards:**
- `--myb-light-blue-soft` (#E0F0FF) background
- Border-radius 12px
- Padding 16px
- Stacked vertically with 12px gap (see Spacing Scale exceptions for rationale)
- Each card: small icon (eye/brain/hand in Navy, 24px) + prompt text (16px, 300, Neutral-5)
- Should feel inviting, not like homework -- rounded corners, generous padding, calm color

**No forward button.** Back button (secondary style: transparent bg, Navy text, Neutral-2 border) to revisit previous screens.

### Post-VR Checklist

**Layout:** Single column, max 640px centered.

**Congrats section:** Heading (40px/28px, Navy, 800) + subtext (24px/20px, Neutral-5, 300)

**Checklist items:**
- Each row: checkbox (24px, rounded-square, `--myb-neutral-2` border) + label (24px, 300, Navy)
- Rows have 12px vertical gap (see Spacing Scale exceptions for rationale) and subtle `--myb-neutral-1` bottom border
- Checked state: checkbox fills with `--myb-primary-blue` + white checkmark inside; label gets `text-decoration: line-through` + color shifts to `--myb-neutral-3`; checkbox micro-bounce animation (250ms spring-like overshoot)
- Each item is `<button>` with `role="checkbox"` and `aria-checked`

**Progress:** "{N} of 6 complete" above checklist. Count number uses `--myb-primary-blue` when N > 0.

**myBlueprint CTA:** Full-width on mobile, max 280px on desktop. `--myb-primary-blue` bg, white text (16px, 800). Opens in new tab with `target="_blank" rel="noopener noreferrer"`.
