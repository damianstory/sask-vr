## Component Inventory

### 1. Name Input

- Standard text input with label "Your first name"
- Placeholder: "Enter your first name"
- Max 30 characters, trimmed whitespace validation
- Border: 1px `--myb-neutral-2`, rounded-xl
- Focus: 3px ring `--myb-primary-blue`
- Inline validation error below input when empty/whitespace-only on blur: red-tinted text "Please enter your first name"
- Updates `firstName` in SessionContext on every keystroke (controlled input)

### 2. Icon Picker Grid

- 3x2 grid of 6 tool icons
- Each cell: rounded-xl, 48px emoji on colored circle centered, tool name label below
- Unselected state: white background, 1px `--myb-neutral-2` border
- Hover state: `shadow-md`, border shifts to `--myb-neutral-3`
- Selected state: `--myb-light-blue` background, 2px `--myb-primary-blue` border, checkmark badge top-right (reuse Screen 2 pattern exactly)
- Focus state: 3px `--myb-primary-blue` ring
- Active state: `scale-[0.98]` press feedback
- Only one icon selectable at a time (radio behavior, not toggle)
- Updates `selectedIcon` in SessionContext on selection

### 3. Task Tag Chips (read-only)

- Section labeled "Your skills"
- 2-3 horizontal chips from `selectedTiles` in SessionContext
- Each chip: rounded-full, `--myb-light-blue` background, `--myb-navy` text, 14px font-300
- Horizontal padding: 12px, vertical padding: 4px
- No interaction (read-only display)
- If no tiles selected (edge case -- navigated directly): show helper text "Go back to pick your tasks" in `--myb-neutral-4`

### 4. Live Card Preview

- 16:9 aspect ratio container (matches 1200x675 output)
- `max-w-lg` on desktop, full-width on mobile
- Rounded-xl with 1px `--myb-neutral-2` border
- Renders as a styled DOM preview (not Canvas) for real-time updates
- Shows: gradient background, "CARPENTER CARD" title, emoji icon on white circle, student name, task tag chips
- Updates in real time as name/icon change
- Empty states within the preview:
  - No name yet: show "Your Name" in 50% opacity white text
  - No icon yet: show empty white circle placeholder

### 5. Download Button

- Full-width below card preview, `max-w-lg`, `min-h-[44px]`, rounded-lg
- Label: "Download Your Card" (from content JSON)
- Enabled state: `--myb-primary-blue` background, white text, font-800
- Hover: `--myb-blue-dark` background
- Disabled state: `--myb-neutral-3` background, white text, `cursor-not-allowed`
- Disabled until both `firstName` (trimmed, non-empty) and `selectedIcon` are set
- On tap: composites card via Canvas API at 1200x675px, triggers download as `carpenter-card.png`

### 6. Post-Download Celebration

- Replaces download button after successful download
- Green checkmark icon + "Your card is saved!" text
- Animate in with `scale-fade-in` keyframe (already exists in globals.css)
- "Continue" button appears below: same style as Screen 2 continue button (`--myb-primary-blue`, font-800, full-width)
- Advances to Screen 6 (VR Prep)

## Interaction States

### Builder Flow (linear)

1. **Initial state**: Name input empty, no icon selected, download disabled, preview shows placeholder content with gradient variant 0
2. **Name entered**: Preview updates name in real time, download still disabled if no icon
3. **Icon selected**: Preview updates icon, gradient recalculates if different tile combo implied. Download enabled if name also provided
4. **Ready state**: Both name + icon set, download button active (primary blue)
5. **Downloading**: Brief loading state (button shows "Saving..." for Canvas render duration)
6. **Downloaded**: Celebration state, continue button appears

### Validation

- Name input: validate on blur, clear error on keystroke
- Whitespace-only names rejected (trim before check)
- Max 30 characters enforced via `maxLength` attribute

### Responsive Layout

- **Desktop/Chromebook (md+ / 768px+)**: Side-by-side layout. Left panel: name input, icon picker, task tags. Right panel: card preview, download button. `gap-8` between panels.
- **Mobile (<768px)**: Single column stacked. Controls above, preview below, download button at bottom. All full-width.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Keyboard operability | Name input: standard tab focus. Icon picker: each icon is a `<button>` with `aria-pressed`. Download button: standard button semantics. |
| Touch targets | Icon cells: 48px minimum dimension. Download/Continue buttons: `min-h-[44px]`. |
| Focus indicators | 3px `--myb-primary-blue` ring on all interactive elements |
| Screen reader | Icon picker buttons: `aria-pressed={isSelected}`, label includes tool name. Download button: disabled state communicated via `disabled` attribute. |
| Reduced motion | Celebration animation respects existing `prefers-reduced-motion` media query rule |
| Contrast | White text on `--myb-primary-blue` (#0092FF): 4.6:1 ratio (AA pass). `--myb-navy` on `--myb-off-white`: 12.5:1 (AAA pass). |
